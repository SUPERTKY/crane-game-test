## 何が変わったか（初心者向け解説）

### 根本的な変更：「手動の数値合わせ」→「メッシュから自動計算」

以前は `HB_Y = -0.22`, `HB1 = { x: 0.10, ... }` のように**人間が数値を手で書いて**ヒットボックスの位置と大きさを決めていました。しかしこの方法だと、メッシュのスケール・回転・オフセットが複雑に絡むため、どうしてもズレます。

今回の修正では、**爪メッシュの実際の形状（バウンディングボックス）から自動的に計算**するようにしました。

```
【修正前】
  手動の数値 → ヒットボックス → ズレる😣

【修正後】
  爪メッシュの形状 → 自動計算 → ピッタリ合う✨
```

---

### 追加した関数の説明

**`meshWorldBoxToBodyLocal(meshRoot)`** — 爪メッシュを囲むワールド空間の箱（AABB）を取得し、物理ボディのローカル座標に変換します。

やっていること：

1. Three.jsの `Box3.setFromObject()` で爪メッシュのワールド空間バウンディングボックスを取得
2. その8つの頂点を、物理ボディの逆回転で「ボディのローカル座標」に変換
3. 変換後の点から新しいバウンディングボックスを作成

**`computeClawShapes(meshRoot, numSegs, padding)`** — 上記のバウンディングボックスをY方向に3分割して、爪の形に近いヒットボックスを生成します。

```
     ┌─────────┐  ← セグメント1（上端：一番太い）
     │         │
     ├────────┤  ← セグメント2（中間：少し細い）
     │        │
     ├───────┤  ← セグメント3（先端：一番細い）
     │       │
     └───────┘
```

先端に向かって25%ずつ細くなる「テーパー」を付けているので、爪の形状に近づきます。

---

### 左右別計算になった理由

元のコードでは左右で同じヒットボックスを使い、Zだけ `HB_Z_SIGN_L / R` で反転していました。しかし左右の爪メッシュは `position` 補正が微妙に異なります（`-1.95` と `-1.85`）。

今回は `clawLMesh`、`clawRMesh` それぞれから個別にヒットボックスを計算するので、左右の微妙な差も自動的に反映されます。

### 調整ポイント（2つだけ）

もし微調整したい場合：

| 定数 | 場所（ファイル上部） | 効果 |
|---|---|---|
| `NUM_CLAW_SEGMENTS` | 29行目 | 分割数（3→5にすればより滑らか） |
| `CLAW_HB_PADDING` | 30行目 | 内側に縮める量（大きくすると当たり判定が小さくなる） |

緑（左）と赤（右）のワイヤーフレームが爪モデルにピッタリ重なっていれば成功です。
